Claude Cowork Setup Checklist

Last updated: March 20, 2026
1. Before You Start
  • Accept your invite — Check your email after registering to activate your license
  • Install Claude Desktop App — Download from MoneyGram portal
  • Sign in with your work email — Click the secure link Anthropic sends you
    Ignore the code page — the email link authenticates you automatically
  • Switch to Cowork mode — Click the Cowork toggle in the desktop app
2. Build Your Workspace
  • Create a master folder "Claude-Work" with these subfolders:
📁 Claude-Work/
├── 📂 ABOUT ME/ — who you are + how you write
│      ├── about-me.md (what you do day-to-day, not your resume)
│      ├── my-voice.md (your tone, phrases you hate, 2–3 writing samples)
│      └── my-rules.md (ask first, show a plan, never delete without approval)
├── 📂 PROJECTS/ — one folder per live project (overview.md in each)
├── 📂 TEMPLATES/ — your best past work (real examples Claude can reference)
└── 📂 OUTPUTS/ — where Cowork saves your files automatically

3. Pick Your Model
  • Choose your model before starting — Once a session starts, the model is locked
Opus
Deep Thinking
Complex reasoning, high-stakes reviews, critical accuracy
PO: Analyzing 50+ pages of user research to find gaps for a PRD
Designer: Reviewing a Design System for accessibility & logic gaps
Marketing: Developing a full-year brand strategy or competitive positioning
Sonnet
Your Default ★
Everyday work — writing, summarizing, brainstorming
PO: Drafting user stories from a rough feature list
Designer: Turning rough requirements into a clear design spec
Marketing: Drafting a personalized outreach email from meeting notes
Haiku
Fast & Simple
Quick questions, formatting, cleanup, speed tasks
PO: Reformatting messy technical notes into clean bullet points
Designer: Generating 20 quick alternative button labels in seconds
Marketing: Brainstorming 20 quick social media headlines
Suggested daily workflow:
• Opus: Plan your day, break down projects, set priorities, and map out tasks
• Sonnet: Execute on the plan — write docs, build decks, draft emails, handle details
• Haiku: Quick ad-hoc tasks — reformat notes, brainstorm names, clean up a table
⚠ Opus burns tokens fast and may hit rate limits. Use it for planning, not all-day work.
⚠ You can't switch models mid-session. Pick your model before you start — you'll need a new session to change it.

Sign in or ask IT

Needs IT: BigQuery, Salesforce, SharePoint — ask IT first
🔧 MCP Servers
Ask IT for credentials
Specialized tools not in the connector list
How: Ask IT for a config snippet and secret key to paste
Use for: External data sources (e.g. OpenCX)
  • Cowork Control Chrome Connector — Let Claude interact with your browser (open pages, read content, fill forms)
How to set it up:
1. Open Claude Desktop → click your Profile icon (bottom-left)
2. Go to Settings → Connectors
3. Find Browser → Control Chrome
4. Click Install — it enables automatically once installed
What this lets you do: Ask Claude to open a webpage, read its contents, fill in forms, click buttons, and take screenshots — all from within Cowork.
  • Cowork Chrome Extension (Claude in Chrome) — Use Claude directly inside your browser while you browse
How to set it up:
1. Open the Chrome Web Store and search for "Claude"
2. Click Add to Chrome to install the extension
3. When prompted, select your organization (MoneyGram)
4. Click Authorize to grant permissions
5. You'll see the Claude icon in your Chrome toolbar, pin it — you're all set
What this lets you do: Highlight text on any webpage and ask Claude about it, get summaries of pages you're viewing, and interact with Claude without leaving your browser.
  • Sign-in Connectors (Okta) — Just sign in with your work account. You can do this yourself.
Settings → Connectors → Browse Connectors → Find your tool → Click Connect → Sign in
Or type "Connect to [Tool]" in the chat and follow the prompts
Examples: Confluence, Jira, Figma, Miro — just click Connect and sign in with your normal work login.
  • Connectors that need IT approval
Examples: BigQuery, Salesforce, SharePoint — ask IT to grant you access first, then connect the same way.
  • MCP Servers — Ask IT for the API key and secret, then follow the steps below
Your IT team will give you two things:
• A config snippet — a block of text to copy-paste
• A secret key — a long string of characters (treat it like a password)
How to set it up:
1. Go to Profile → Settings → Developer → Edit Config
2. Paste the config snippet IT gave you
3. Replace "your-api-key-here" with the secret key IT provided
4. Save and restart Claude Desktop
Example config snippet (OpenCX):
{ "mcpServers": { "opencx": { "command": "npx", "args": ["-y", "@opencx/mcp"], "env": { "OPENCX_API_KEY": "your-api-key-here" } } } }
⚠ Treat API keys and secret keys like passwords. Never share in Slack, email, or docs.
6. Your Standard Workflow
Follow this flow every time you open Cowork.
  • 1. Open Cowork and select your Claude-Work folder
  • 2. Pick your model — Opus for planning, Sonnet for doing, Haiku for quick tasks
  • 3. Keep your prompt short — Claude already reads your files. Example: "Read my files. I need to draft a status update for the Rewards project."
  • 4. Review Claude's plan — Claude will show you a plan before it starts. Approve it, adjust it, or ask questions.
  • 5. Review the output — You're the expert, Claude is the assistant. Ask for changes: "Make it shorter" or "Change the tone to be more casual."
  • 6. Save your work before closing — Ask Claude to save deliverables to your OUTPUTS folder, Jira, or Confluence.

8. Stop Doing
  • Writing long prompts for every task → Keep it short. Claude reads your files.
  • Skipping the folder setup → Set up Claude-Work once. Reuse it every session.
  • Using Chat when you need Cowork → Chat can't read your files or connect to your tools.
  • Expecting Claude to know you without files → Fill in your ABOUT ME folder. That's how Claude learns you.
